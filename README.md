# Typewall

A browser-based stylized text image generator. Type text, pick a size and font,
apply spray-paint, shadow, or outline effects, add backgrounds, overlays, and
watermarks, then download. Use the Remix button to randomize unlocked
dimensions for rapid variation. Generate multiple combinations in bulk and
download them as a ZIP.

All rendering happens locally in your browser. No server, no network requests,
no data leaves your machine.

## Quick start

```bash
npm install
npm run dev
```

Open `http://localhost:5173` in your browser.

## Build

```bash
npm run build
```

Output is written to `dist/`. The build automatically runs `gen:manifest` first
to regenerate the asset manifest and create the `public/` symlinks.

```bash
npm run gen:manifest
```

Run this manually after adding or removing files in `fonts/`, `backgrounds/`,
`overlays/`, `watermarks/`, or `textmasks/`. It writes
`src/assets/manifest.generated.ts` and creates `public/` symlinks so Vite can
serve the asset directories at `/`.

## How assets and fonts work

All assets are bundled locally. No CDN or external URLs are fetched at runtime.

- **Fonts**: TTF/OTF files in `fonts/`. The UI fonts (Archivo, Inter, JetBrains
  Mono) are included via `@fontsource` packages bundled into the JS output.
- **Backgrounds, overlays, watermarks, textmasks**: image files in their
  respective root directories, served via `public/` symlinks.
- **User uploads**: images you upload (custom backgrounds, watermarks, etc.) are
  held in memory only. They are not persisted between sessions.

`public/` is git-ignored and regenerated by `gen:manifest` on every build.

## Testing

```bash
npm test
```

Runs all Vitest suites including engine unit tests, RNG determinism tests, and
an end-to-end headless render smoke test.

## Meme mode

Enable the **Meme** checkbox in the top toolbar (next to Mix fonts). When active,
the CONTENT section of the control rail replaces the single text field with a
**Meme style** switch (Overlay or Bars), a TOP CAPTION field, and a BOTTOM
CAPTION field. Captions are always rendered uppercase using Anton, the bundled
OFL substitute for the proprietary Impact font.

- **Overlay**: white caption text with a thick black outline drawn over the image.
- **Bars**: black letterbox bands at the top and bottom of the image with white
  text inside.

The Remix button is meme-aware: it draws fonts only from a bold/condensed subset
(Anton, Archivo Black, Bebas Neue, Staatliches, Bungee), keeps text white and
outline black, and alternates between Overlay and Bars. Note: in Batch mode the
text axis controls `content`, not captions, so caption text is not a batch axis.

## Deploy to GitHub Pages

Push to `main`. The `.github/workflows/deploy.yml` workflow runs
`npm ci && npm test && npm run gen:manifest && npm run build` and publishes `dist/` to
GitHub Pages. The Vite config uses `base: './'` so the app works under a repo
subpath.

To enable Pages for your fork: go to Settings > Pages, set the source to
"GitHub Actions".

### Origin

Typewall began as a Python/Pillow command-line tool and was rewritten as a
fully client-side TypeScript web app. The spray-paint effect is a port of that
original stencil algorithm.

## Project Overview

Typewall is a browser-based stylized text image generator. The web app (React + Vite + TypeScript) lets you compose text with backgrounds, overlays, spray-paint effects, watermarks, and shuffle randomization, then download single images or bulk ZIP batches.

It began as a Python/Pillow CLI; the spray-paint effect is a port of that original stencil algorithm. The Python source is no longer kept in the repo.

## Repository Structure

## Engine Architecture

The `src/engine/` pipeline is the authoritative render path used by both the
browser preview and the worker pool. It contains no browser-only globals and
runs cleanly under Node (via @napi-rs/canvas in tests).

Pipeline order: background -> overlay -> per-line text layers -> composite ->
spray paint -> text mask -> center onto canvas -> watermark.

Key modules:
- **pipeline.ts**: `render(config, target, assets)` -- top-level orchestrator
- **types.ts**: `RenderConfig`, `defaultConfig()`
- **spray-paint.ts**: 5-phase stochastic spray effect (erosion, cloud, threshold, overspray, drips)
- **font-fit.ts**: dynamic font sizing to fill available canvas area
- **compositor.ts**: per-line layer compositing with alignment + slant
- **surface.ts**: `Surface` type (OffscreenCanvas | HTMLCanvasElement | napi canvas)
- **rng.ts**: seeded RNG for deterministic rendering
- **meme-layout.ts**: meme caption layout (Overlay/Bars); config block `config.text.meme` (`enabled`, `style`, `topText`, `bottomText`)
- **letterSpacingPct**: tracking on `TextConfig`, normalized (px =
  letterSpacingPct * fontPx), clamped [-0.05, 0.4]. Applied in `text-render.ts`
  via `ctx.letterSpacing` with a per-glyph fallback for canvases lacking it.

## Assets: Fonts and Images

All assets are bundled locally (no CDN, no remote fetch at runtime). Fonts are
served from `fonts/` via a public/ symlink created by gen:manifest. The
@fontsource packages (Archivo, Inter, JetBrains Mono) are listed under
`dependencies` because they ship in the production bundle.

User-uploaded assets (backgrounds, watermarks, etc.) are held in memory only
and never persisted. The manifest lists only bundled assets.

## Favicon and App Icons

The tracked sources live in `src/assets/`: `favicon.ico`, `favicon-16x16.png`,
`favicon-32x32.png`, `apple-touch-icon.png`, `android-chrome-192x192.png`,
`android-chrome-512x512.png`, and `site.webmanifest`. `gen-manifest.ts`
(`ICON_FILES`) copies each to `public/` at build time so Vite emits them into
`dist/`. `index.html` references them by relative path (for `base: './'` subpath
deploys); `site.webmanifest` icon `src` values are likewise relative. To add or
change an icon, drop the file in `src/assets/` and add it to `ICON_FILES`.

## Deploy

Push to `main` triggers `.github/workflows/deploy.yml`, which runs
`npm ci && npm test && npm run gen:manifest && npm run build` and publishes `dist/` to
GitHub Pages. The Vite config uses `base: './'` for subpath compatibility.

## Spray Paint Effect

5-phase stochastic stencil effect ported from the original Python/Pillow CLI:
1. Edge Erosion: coarse noise subtraction for irregular borders
2. Probability Cloud: dilation + blur for soft gradients
3. Stochastic Thresholding: noise-based pixel decisions
4. Overspray Layer: sparse splatters using blob and grit noise
5. Drip Generation: random vertical lines from text bottom edges

Config fields: `text.sprayPaint.enabled`, `text.sprayPaint.overflow`.
Scale factor = fittedMaxFontPx / 80 (mirrors reference FONT_SIZE_REFERENCE).

## Text Line Breaking

Manual line breaks come from either a literal newline (Enter) or `//`:
```
"Education//is important but//big biceps//are importanter"
```
Both separators may be mixed. Without any separator, text wraps automatically
based on calculated line length. See `src/engine/wrap.ts` (`splitLines`).

## Vibe Presets

Remix supports curated "vibe" presets (`src/shuffle/presets.ts`): brutalist,
y2k, vaporwave, newspaper. A preset is a `VibeBias` that constrains the ranges
and pools each shuffle dimension rolls from, so Remix stays on-theme while still
varying. Selected via the Vibe dropdown in `ShuffleBar`; stored as
`activeVibe`. `none` = unbiased random. Locks still apply.

## Remix Color Contrast

When the Color dimension is unlocked, Remix biases the text color toward palette
entries that stay readable against the background. `src/shuffle/contrast.ts`
computes WCAG contrast ratios; the roll keeps only palette colors at or above
`MIN_CONTRAST_RATIO` (3:1), falling back to the single highest-contrast color
when none qualify. The background reference is `background.color` for a solid
background, or the average color of the background image (computed on the main
thread in `PreviewCanvas`, cached in the store as `bgAvgColor` keyed by asset
id, passed via `ShuffleOpts.backgroundColor`). A transparent background, or an
image whose average is not yet computed, disables filtering (unbiased roll).

`top-middle`, `bottom-middle`, `left-middle`, `right-middle`.
